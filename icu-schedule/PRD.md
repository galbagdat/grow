# PRD — ICU Kazakhstan 2026/27: Панель запуска потока

**Версия:** 1.2
**Дата:** 2026-07-07
**Автор:** Галия Багдат / Claude

## 1. Назначение
Единое офлайн веб-приложение (один HTML-файл, без сервера) для планирования и запуска потоков ICU в 9 городах на учебный год 2026/2027: старт 15 сентября 2026 в Актау.

## 2. Ключевой алгоритм расписания
- 9 потоков: Актау, Астана, Алматы, Атырау, Актобе, Шымкент, Караганда, Павлодар, Онлайн.
- 8 модулей на поток, по 2 дня, цикл модуля — 35 дней (5 недель).
- Старты потоков распределены со сдвигом 4 дня друг от друга, начиная с 15 сентября 2026 (Актау).
- 9 тренеров-ротаторов (Галия Багдат, Гульмира Чупекова, Анар, Тогжан, Гульмира Сыздыкова, Эльмира, Элина, Дания, Михаил) распределены по формуле `тренер = (индекс_города + номер_модуля) mod 9`.
- **Проверено программно: ни один тренер не ведёт модули в двух городах в одну и ту же дату.** Минимальный интервал между сессиями одного тренера — 31 день (комфортно для перелётов).
- Итог: 73 сессии на год (72 модуля + 1 доп. сессия командного коучинга), файл `schedule_2026.json` встроен в приложение.
- Локальные тренеры-ассистенты Шымкента (Рахат, Айгуль) остаются в справочнике тренеров как дополнительные/локальные — не участвуют в основной ротации во избежание конфликтов.
- **НЛП-тренеры (Хованская, Длужневский) полностью убраны из программы.**
- **Командный коучинг (весна 2027):**
  - **Атырау, Модуль 7 (25–26 апреля 2027)** — тренер заменён на **Данагуль**, руководителя офиса Атырау; сессия помечена как «Командный коучинг».
  - **Актау** — стандартный Модуль 4 приходится на 29–30 декабря (зима), поэтому вместо переноса даты добавлена **отдельная доп. сессия «Командный коучинг» 23–24 марта 2027**, которую ведёт новый тренер — руководитель офиса Актау (имя пока не указано, поле готово к заполнению во вкладке «Тренеры»).

## 3. Роли и доступ
- **Администратор** (Галия) — вход по PIN (по умолчанию `ICU2026`). Единственная роль с доступом к вкладкам «Отчёты и Excel» и «Финансы».
- **Представитель города** — выбор города без пароля, доступ к своему городу (студенты, карточка представителя), просмотр расписания/тренеров/рейсов/шаблонов/обратной связи.
- **Тренер** — выбор своего имени из списка без пароля. **Видит только вкладку «Моё расписание»** — сессии, где преподаёт лично; остальные разделы скрыты.
- Важно: это разграничение на уровне интерфейса (файл открыт локально), а не шифрование — подходит для доверенной внутренней команды, не для публикации в интернете.

## 4. Разделы приложения
1. **Дашборд** — сводная готовность по всем городам, ближайшие модули.
2. **Расписание 2026/27** — 73 сессии (72 модуля + 1 доп. сессия командного коучинга), фильтры, экспорт в Excel; тренер видит только свои сессии («Моё расписание»).
3. **Тренеры** — карточки тренеров (9 основных ротаторов + 2 локальных Шымкента + 2 командного коучинга = 13), контакты редактируются администратором.
4. **Города и роли** — 9 карточек: представитель, контакты, заметки, прогресс набора, статус готовности.
5. **Студенты** — CRUD по студентам (ФИО, телефон, email, статус, прогресс по 8 модулям), аналитика по городу.
6. **Перелёты** — справочник авиакомпаний/частоты рейсов в города программы (индикативно, с пометкой проверять актуальное расписание).
7. **Отчёты и Excel** (только админ) — сводная готовность; полный экспорт .xls с отдельным листом на каждый город (расписание + студенты) плюс сводные листы.
8. **Финансы** (только админ) — стоимость одного модуля (по умолчанию **240 000 ₸**, редактируется), полная программа = 8×цена; потенциальный доход (по целям набора) и признанный доход (по факту пройденных модулей); диаграмма дохода по городам; **папка «Отчёты посещений групп»** — отметка присутствия по каждому модулю/городу с выгрузкой в Excel.
9. **Обратная связь** — QR-код (офлайн-генерация, без интернета) со ссылкой на WhatsApp представителя города для отзыва студента; ручной журнал отзывов (оценка звёздами + комментарий), доступен админу (все города) и представителю (свой город).
10. **Рассылка** — WhatsApp (wa.me) и Email (mailto) со связкой шаблонов и подстановкой имени/города/даты/тренера; отправка вручную, от аккаунта пользователя.
11. **Шаблоны** — PowerPoint-презентация запуска потока, встроена в файл как base64 (скачивание без интернета); ссылка на полный Excel-отчёт по городам.

## 5. Хранение данных
- `localStorage` браузера: города/представители, студенты, PIN администратора, шаблоны сообщений, контакты тренеров.
- Данные привязаны к конкретному браузеру/устройству — при необходимости общего доступа для всех представителей нужен либо общий компьютер, либо ручной экспорт/импорт (в v1 — только экспорт в Excel; импорт можно добавить в v1.1).

## 6. Известные ограничения / бэклог

- [ ] Импорт студентов из Excel обратно в приложение.
- [ ] Реальные контакты тренеров и представителей — сейчас поля пустые, заполняются вручную в приложении (без телефона представителя QR-код обратной связи не формируется).
- [ ] Данные о рейсах — индикативные, не через живой API; рекомендовано ссылаться на airastana.com / scat.kz / flyarystan.com.
- [ ] Синхронизация между устройствами (сейчас только localStorage конкретного браузера).
